      ### Server Actions
      The contact form is implemented using a <Primary>**typed Server Action**</Primary>, validated on the server with Zod.
      The goal was to explore the constraints of this model: how to handle errors, how to type everything consistently, and how to ensure scalability.

      ### App Router
      The entire project is built using the App Router from <Primary>**Next.js 16**</Primary>.
      This allowed me to clearly identify:

      - The **advantages**: shared layouts, Server Components
      - The **limitations**: hydration constraints, global state management
      - Their **impact** on how to <Primary>**structure**</Primary> a project
